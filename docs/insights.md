# Insights and Design Notes

## Insight 1: Accessibility is a product feature

Strong accessibility practices (semantic controls, focus visibility, keyboard parity, reduced motion support) improve both usability and engineering quality.

## Insight 2: Performance protects first impressions

Large media and heavy graphics should be intentionally staged. Progressive loading and lightweight fallbacks reduce bounce risk and improve perceived quality.

## Insight 3: Narrative clarity drives conversion

Project cards become more persuasive when they quickly show problem, process, constraints, and measurable outcomes.

## Insight 4: Verification-first iteration scales better

Incremental enhancements with continuous diagnostics checks reduce regression risk in live portfolio updates.
